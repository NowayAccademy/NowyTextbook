# リモートと共有（GitHub）
`clone` / `push` / `pull` と、作業開始前の pull の重要性です。

## 本章の目標

本章では以下を目標にして学習します。

- GitHub が**共有のリモートリポジトリ**としての役割だと説明できること
- `git clone` / `git push` / `git pull` の意味を区別できること
- **作業開始前に pull する**理由を述べられること

## 1. リモートとは

手元のリポジトリとは別に、**サーバー上のリポジトリ**（例：GitHub）と同期します。  
チームではここを**正**にすることが多いです。

## 2. よく使うコマンド

```bash
git clone https://github.com/org/repo.git
cd repo
# 作業…
git pull origin main
git push origin main
```

- `clone` … リモートから**丸ごと複製**  
- `pull` … リモートの更新を**取り込む**（fetch + merge など）  
- `push` … 手元のコミットを**リモートに送る**  

> **ポイント**  
> **作業開始前に `pull`** しておくと、古い履歴の上に作業してコンフリクトや無駄なマージが増えるのを防げます。

## 3. 事前準備

受講前に **GitHub アカウント**を作成しておくとスムーズです（ハンズオンで使用）。

## まとめ

- GitHub は**共有と可視化**のハブである  
- `pull` で取り込み、`push` で送る  
- **こまめな pull** が衝突を減らす
