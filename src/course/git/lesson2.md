# Gitの基本操作と概念
3つの状態、`init` / `status` / `add` / `commit` とコミットメッセージの心得です。

## 本章の目標

本章では以下を目標にして学習します。

- 作業ディレクトリ・ステージング・リポジトリの違いが分かること
- `git init` / `git status` / `git add` / `git commit` の流れを説明できること
- 実務的なコミットメッセージの書き方の型を持てること

## 1. Git の3つの状態（イメージ）

| 場所 | 意味 |
| --- | --- |
| 作業ディレクトリ | いま編集しているファイル |
| ステージング（index） | **次のコミットに載せる**変更の待機場所 |
| リポジトリ（.git） | **コミットとして記録**された履歴 |

## 2. 基本コマンド

```bash
git init
git status
git add .
git commit -m "Add login form"
```

- `git add` … ステージングに上げる（ファイル単位・パッチ単位も可能）  
- `git commit` … ステージングの内容を1コミットとして記録  

> **ポイント**  
> `git add .` は手軽ですが、**意図しないファイルまで載る**ことがあります。`git status` で毎回確認する習慣をつけましょう。

## 3. コミットメッセージ（実務の型）

- **何をしたか**が一行で分かる（英語なら命令形、日本語なら「〜を追加」など）  
- 必要なら本文で**理由・背景**を書く（なぜそうしたか）  

悪い例：`fix` / `update` だけ  
良い例：`Fix null check on user profile API`

## まとめ

- 変更は **add → commit** で履歴に残る  
- `status` で**今どこにいるか**を常に確認する  
- メッセージは**未来の自分とチーム**への手紙である
