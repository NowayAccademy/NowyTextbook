# Gitとは
なぜバージョン管理が必要か、Git の立ち位置を理解します。

## 本章の目標

本章では以下を目標にして学習します。

- バージョン管理がないと何が困るか説明できること
- Git が**変更履歴を追跡する**ツールだと言えること
- チーム開発との関係をざっくり述べられること

## 1. なぜ Git が必要か

ソースコードは日々変わります。**いつ・誰が・何を変えたか**が追えないと、次のような状況で詰みます。

- 昨日まで動いていたのに、今日動かない  
- バグが入ったが、**どの変更が原因か**分からない  
- 複数人で同じファイルを編集して**上書き**してしまう  

> **ポイント**  
> Git は「**時系列のスナップショットの鎖**」として履歴を持ちます。コミット単位で戻したり、差分を比較したりできます。

## 2. バージョン管理の基本概念

- **リポジトリ** … 履歴を保管する場所（フォルダに `.git` ができる）  
- **コミット** … ある時点の状態を記録するまとまり  
- **ブランチ** … 履歴の別ライン（機能ごとに分けるなど）  

## 3. 他ツールとの違い（ざっくり）

Git は**分散型**が特徴で、各自の手元にも履歴のコピーを持てます。  
集中型の VCS と比べ、**オフライン作業**や**マージ文化**がしやすいです。

## まとめ

- 変更履歴がないと**トラブル時に詰む**  
- Git は履歴を**コミット**で積み上げる  
- チーム開発の**土台**になる
