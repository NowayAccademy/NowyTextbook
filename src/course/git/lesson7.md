# ハンズオン：PRとリバート体験
ブランチ・PR・マージ・`revert` を一連で体験します。

## 本章の目標

本章では以下を目標にして学習します。

- ブランチを切ってコミットし、GitHub 上で **PR を作成**できること
- マージ後に **`git revert`** で意図的に変更を打ち消せること
- 共同作業の**ふりをした手順**を自分の言葉で説明できること

## 1. 想定シナリオ

1. リポジトリを **clone**  
2. 作業用ブランチを作成  
3. ファイルを編集して **add / commit**  
4. **push** し、GitHub で **Pull Request** を作る  
5. レビュー（想定）のあと **マージ**  
6. 問題があった変更を **`git revert`** で打ち消す  

> **ポイント**  
> **事前に GitHub アカウント**を作成しておいてください。SSH と HTTPS のどちらで clone するかは環境に合わせます。

## 2. revert で何が起きるか

マージコミットを revert する場合は **`-m` オプション**が必要になることがあります。ハンズオンでは講師の手順に従ってください。

## 3. 振り返り

- PR には**説明文**を書く  
- revert は**履歴を残す**取り消しである  

## まとめ

- PR は**変更の窓口**、マージは**本線への統合**である  
- `revert` で**安全に巻き戻し**の体験ができる  
- 手を動かしたあと**図にして説明**できると定着する
