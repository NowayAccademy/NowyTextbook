# ブランチとマージ
ブランチの意味、`branch` / `checkout`（または `switch`）、コンフリクト解消の流れです。

## 本章の目標

本章では以下を目標にして学習します。

- ブランチが**履歴の別ライン**だと説明できること
- ブランチ名の**命名の例**（feature/ など）を知っていること
- マージでコンフリクトが出たときの**解消手順の骨子**が分かること

## 1. ブランチとは

**main**（旧 master）を壊さず、機能開発や修正を**並行**するための仕組みです。

```bash
git branch feature/login
git checkout feature/login
# または: git switch feature/login
```

## 2. マージ

作業ブランチを main に取り込むと、履歴が**合流**します。

```bash
git checkout main
git merge feature/login
```

## 3. コンフリクト

同じ行を別ブランチで変えていたとき、Git が**自動マージできず**止まります。

1. コンフリクトファイルを開く  
2. `<<<<<<<` `=======` `>>>>>>>` を手で解消する  
3. `git add` してから `git commit`（マージコミットになることも）  

> **ポイント**  
> コンフリクトは**悪ではない**。早めに小さくマージする、コミュニケーションを取る、で頻度と大きさを抑えます。

## まとめ

- ブランチで**並行作業**を分離する  
- マージで**変更を統合**する  
- コンフリクトは**手で意図を選んで解消**する
