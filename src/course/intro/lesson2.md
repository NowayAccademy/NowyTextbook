# 環境構築のフォロー
事前に用意した開発環境で詰まった点を解消します。

## 本章の目標

本章では以下を目標にして学習します。

- エディタ・ブラウザ・ターミナルなど、学習に必要な道具が揃っている状態にできること
- エラーメッセージを見て、検索・質問の材料にできること

## 1. この時間の目的

**基本的な環境構築は事前に完了**してもらう想定です。  
このレッスンは、個人ごとに**どうしても解消できなかった部分**をフォローする時間です。

> **ポイント**  
> 「自分だけつまずいている」と感じやすいテーマです。ログや画面のスクリーンショットを残しておくと、手伝ってもらいやすくなります。

## 2. よくあるつまずき（例）

| 分類 | 例 |
| --- | --- |
| インストール | 権限不足、パスが通らない、古いバージョンが残っている |
| エディタ | 拡張機能、文字コード、改行コード |
| ブラウザ | キャッシュ、別ブラウザでの動作差 |

## 3. 質問するときに伝えるとよいこと

- **OS**（Windows / macOS など）とバージョン  
- **やろうとした操作**と**起きた現象**（期待した結果・実際の結果）  
- **表示されたメッセージ**（そのままコピーできるとベター）

## まとめ

- 環境構築は**事前準備が基本**、本編は**個別フォロー**である  
- エラーは**メッセージごと共有**すると解決が早い  
- 解決した内容は**メモしておく**と再発時に楽である
