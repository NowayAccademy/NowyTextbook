# デバッグ（開発者ツール）
Elements・Console・Sources・Network の基本を押さえます。

## 本章の目標

本章では以下を目標にして学習します。

- Chrome（または同等ブラウザ）の開発者ツールを開けること
- Elements で DOM とスタイルを確認できること
- Console でログとエラーを読めること
- Sources でブレークポイントを置くイメージを持てること
- Network でリクエストの成否をざっくり見られること

## 1. 開発者ツールの開き方

- Windows: `F12` や `Ctrl + Shift + I`  
- macOS: `Cmd + Option + I`  

## 2. Elements

- **DOM ツリー**の実際の状態を見る（HTML ソースと差があることもある）  
- スタイルの**当たり先・打ち消し線**を確認できる  

> **ポイント**  
> 「思った色にならない」は**詳細度**や**継承**の問題であることが多いです。

## 3. Console

- `console.log` の出力  
- **エラーメッセージ**（ファイル名・行番号の手がかり）  

## 4. Sources

- JavaScript に**ブレークポイント**を置き、実行を止めて変数を覗く  
- ステップ実行で**処理の流れ**を追う  

## 5. Network

- **GET/POST** のステータスコード（200, 404, 500 など）  
- レスポンスが JSON か HTML か、**失敗しているか**の確認  

## まとめ

- Elements＝**見た目と DOM**、Console＝**ログとエラー**  
- Sources＝**実行の一時停止と追跡**  
- Network＝**通信の成否**
