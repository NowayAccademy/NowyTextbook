# 変数・データ型と基本構文
前章でJavaScriptを読み込む準備ができました。  
本章では、プログラムに「データを記憶させる」ための変数と、データの種類（データ型）、そして「もし〇〇だったら」という条件分岐の基本を学習します。  

## 本章の目標  

本章では以下を目標にして学習します。  

- `const` と `let` の違いを理解し、実務のルールで使い分けられること  
- 基本的なデータ型（文字列、数値、真偽値）を説明できること  
- `if` 文と厳密等価演算子（`===`）を使って、バグの起きにくい条件分岐が書けること  

## 1. 変数（データを記憶する箱）  
JavaScriptで何かしらのデータを扱う際、そのデータを一時的に入れておく「箱」を用意します。これを**変数**と呼びます。  

変数を宣言する（箱を作る）には、主に **`const`** と **`let`** の2つのキーワードを使います。  

### `const`（定数：再代入できない）  
一度箱に入れたデータを、後から別のデータに書き換える（再代入する）ことができない宣言方法です。  
**現代のJavaScript開発では、9割以上この `const` を使用します。**  

```javascript
const myName = "山田";
console.log(myName); // "山田" と表示される

// ❌ エラーになる！（constは上書き禁止）
myName = "佐藤";
```

let（変数：再代入できる）  
後から中身のデータを書き換える予定がある場合に使用します。 
```javascript 
let age = 20;
console.log(age); // 20

// ⭕️ letは上書き可能
age = 21; 
console.log(age); // 21
```  
`var`は絶対に書きません  
古い記事や教材を見ると`var`という変数宣言が出てきますが、現代の現場では「意図しない上書き」によるバグを引き起こす危険な仕様のため、使用禁止が常識です。  
「基本はすべて`const`で書き、後から書き換える必要があるものだけ let に変える」と覚えておきましょう。  

## 2. データ型（データの種類）  
箱に入れるデータには「種類」があります。  
コンピュータに正しく計算させるためには、データの種類を意識する必要があります。  
### 1.文字列 (String)  
テキストデータです。  
シングルクォーテーション（'）かダブルクォーテーション（"）で囲みます。  
```javascript
const greeting = "こんにちは";
const numberText = "100"; // クォーテーションで囲むと「文字」扱いになる
```  
### 2.数値 (Number)  
計算用の数値です。クォーテーションで囲まずにそのまま書きます。  
```javascript
const price = 1000;
const taxRate = 0.1;
```  
### 3.真偽値 (Boolean)  
「正しいか（`true`）」か「間違いか（`false`）」の2通りのみを持つデータ型です。  
この後学ぶ「条件分岐」で非常に重要な役割を持ちます。  
```javascript
const isAdult = true;   // 大人である
const isMember = false; // 会員ではない
```  

## 3. 条件分岐（if文）  
「もし〇〇だったら、Aの処理をする。  
そうでなければBの処理をする」というプログラムの基本構造を作ります。  

```javascript
const point = 85;

// () の中に条件式を書く
if (point >= 80) {
  console.log("素晴らしい！合格です。");
} else if (point >= 60) {
  console.log("惜しい！あと少しです。");
} else {
  console.log("不合格です。再テストを受けてください。");
}
```  
比較演算子  
条件を作るための記号です。  
- A >= B: AはB以上  
- A === B: AとBは等しい  
- A !== B: AとBは等しくない  
===（イコール3つ）を使う理由  
JavaScriptには ==（イコール2つ）という比較演算子もありますが、現場では必ず ===（厳密等価演算子）を使います。  
== は勝手に気を利かせて、数値の 1 と文字列の "1" を「同じだね！」と判定してしまい、重大なバグの原因になります。  
=== は「データの型（種類）まで完全に一致しているか」を厳密にチェックしてくれます。  

## まとめ  

- 変数の宣言は、原則として再代入不可の`const`を使い、変更が必要な場合のみ`let`を使う。  
- 古い記法である`var`はバグの温床になるため使用しない。  
- データには「文字列（String）」「数値（Number）」「真偽値（Boolean）」などの種類がある。  
- 条件分岐は`if`文を使い、比較には型までチェックする === を必ず使用する。
