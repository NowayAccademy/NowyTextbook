# その他よく使う実装（UIパターン）
モーダル・タブ・活性制御など、現場で繰り返し出るパターンの考え方です。

## 本章の目標

本章では以下を目標にして学習します。

- モーダル表示の「見せる／隠す」を class や style で切り替えられるイメージを持てること
- タブ UI が「表示ブロックの切り替え」だと説明できること
- ボタンの disabled など**活性制御**の必要性を述べられること

## 1. モーダル（オーバーレイ）

典型は次の組み合わせです。

- 半透明の**背景レイヤー**（オーバーレイ）  
- 手前の**ダイアログ用の箱**  
- **表示フラグ**（開いているか）に応じて CSS で `display` や `class` を切り替える  

```html
<div id="modal" class="modal is-hidden">
  <div class="modal__content">内容</div>
</div>
```

```javascript
// イメージ: is-hidden を付け外しする
modal.classList.remove("is-hidden");
```

> **ポイント**  
> アクセシビリティでは **フォーカスの管理**（モーダル外にフォーカスが逃げない等）も重要です。まずは動作の骨格を理解し、段階で足していきましょう。

## 2. タブ

- タブ見出しをクリック → **対応するパネルだけ表示**、他は非表示  
- どのタブがアクティブかを **data 属性**や **class** で表すことが多い  

## 3. 活性制御

- 送信中はボタンを **`disabled`** にして二重送信を防ぐ  
- 入力不足のときは送信ボタンを押せなくする、など  

## まとめ

- モーダル・タブは**見た目の出し分け**が中心である  
- **classList** や **style** で状態を切り替える  
- **disabled** などでユーザー操作を制御する
