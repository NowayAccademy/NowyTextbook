# JavaScriptの役割と記述方法
HTML編でWebページの「構造」を、CSS編で「装飾（見た目）」を作る方法を学びました。  
本コースからは、そのWebページに「動き」と「頭脳」を与えるJavaScriptについて学習します。  

## 本章の目標  

本章では以下を目標にして学習します。  

- HTML、CSS、JavaScriptそれぞれの役割分担を説明できること  
- JavaScriptを読み込む3つの方法を知り、実務で使われる手法を選択できること  
- 外部ファイルを読み込む際の `defer` 属性の重要性を理解すること  

## 1. JavaScriptとは何か？  
JavaScriptは、Webページに「動き」や「対話性（インタラクティブ性）」を追加するためのプログラミング言語です。  

例えば、「ボタンを押したらポップアップが出る」「画像が数秒ごとに自動で切り替わる」「パスワードが短すぎたらエラーメッセージを出す」といった処理は、すべてJavaScriptが担当しています。  
現代のWeb開発において、JavaScriptは単なる「動きの追加」にとどまらず、裏側でデータをやり取りしたり、Webアプリ全体を制御する「頭脳」として欠かせない存在になっています。  

## 2. JavaScriptを適用する3つの方法  
CSSと同じように、JavaScriptのコードをHTMLに反映させるには主に3つのアプローチがあります。  

### 1. 外部ファイルとして読み込む（実務の標準）  
拡張子が `.js` の専用ファイル（例：`main.js`）を作成し、HTMLの `<script>` タグで読み込みます。読み込みは `src` 属性にパスを記述します。  
コードが長くなってもHTMLが散らからず、保守性が高まるため、現場では基本的にこの方法を使用します。  

```html
<head>
  <script src="./js/main.js" defer></script>
</head>
```  

### 2. `<script>`タグ内に書く  

HTMLファイル内に`<script>`タグを用意し、その中に直接JavaScriptを記述します。  
ページ固有の短い処理を書く場合や、学習の初期段階でサクッと動きを確認したい場合に使われます。  

```html
<body>
  <h1>NOWAYカフェへようこそ</h1>
  
  <script>
    // ここにJavaScriptのコードを書く
    console.log("ページが読み込まれました！");
  </script>
</body>
```  

`getElementById`は **id が一意**であることが前提です。  

### 3. インラインで直接書く（非推奨）  
HTMLのタグに onclick などの属性を直接追加して記述します。  
HTML（構造）とJavaScript（動き）が密接に混ざりすぎてしまい、後から修正するのが非常に困難になるため、通常のコーディングでは原則として使用しません。  
```html
<button onclick="alert('クリックされました！')">送信する</button>
```
- `querySelector` / `querySelectorAll` … CSS セレクタ風に取得  
- `getElementsByClassName` … クラス名で複数取得  

## 実務の鉄則 defer 属性  
外部ファイル（.js）を読み込む際、現場で必ず使うのが`defer`（ディファー）属性です。  
```html
<script src="main.js" defer></script>
```
ブラウザはHTMLを上から順に読み込みますが、`<script src="...">`を見つけると、「JavaScriptの読み込みと実行が終わるまで、HTMLの描画をストップする」という性質があります。これにより、画面が真っ白な時間が長くなってしまいます。  
defer 属性をつけると、ブラウザに「HTMLの読み込み（画面の描画）を優先して進めておいて、裏側でJavaScriptを準備し、HTMLが全部読み終わってからJSを実行してね」と指示することができます。  
これにより、ページの表示速度が上がり、ユーザー体験（UX）が向上するため、モダンなWeb開発では必須の設定となっています。  

## まとめ  

- JavaScriptはWebページに「動き」と「頭脳」を与えるプログラミング言語である。  
- HTML（構造）、CSS（装飾）、JavaScript（動き）で役割を完全に分離するのが鉄則。  
- 外部ファイル（.js）を作成し、HTMLの`<script>`タグで読み込むのが実務の基本。  
- ページの表示速度を落とさないため、JSの読み込みには必ず defer 属性をつける。
