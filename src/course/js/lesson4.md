# 同期処理と非同期処理の概念
待つ／待たないの違いと、Web での API 通信の入り口を整理します。

## 本章の目標

本章では以下を目標にして学習します。

- 同期処理が「上から順に完了まで待つ」流れだと説明できること
- 非同期で「時間のかかる処理を待たずに次へ進む」イメージを持てること
- GET と POST の違いを用途で言えること
- JSON でデータをやり取りすることが多いと理解すること

## 1. 同期と非同期

- **同期** … ある処理が終わるまで**次の行に進まない**  
- **非同期** … 処理を依頼したら**先に進み**、終わったらコールバックや `Promise` などで結果を受け取る  

ネットワーク通信やタイマーは、**非同期**で扱うのが普通です。

> **ポイント**  
> 「非同期＝速い」ではなく、**UI を固まらせない**・**サーバーの応答を待つ間も他の処理を進められる**、という意味合いが強いです。

## 2. GET と POST（HTTP のメソッド）

| メソッド | ざっくり用途 |
| --- | --- |
| **GET** | **取得**（検索、一覧、詳細の閲覧など）。URL に条件を載せることも多い |
| **POST** | **登録・更新の依頼**など、サーバー側の状態を変える操作に多い |

実務では **REST** らしく `PUT` / `PATCH` / `DELETE` も使いますが、入門では GET/POST の区別が土台です。

## 3. JSON と Ajax / fetch

- **JSON** … キーと値のペアでデータを表す**テキスト形式**。API の送受信でよく使われます。  
- **Ajax** … ページ全体を切り替えずに**部分的にサーバーと通信**する考え方の総称（歴史的な呼び方）。  
- 現代のブラウザでは **`fetch` API** で HTTP リクエストを送る例が多いです。  

```javascript
// イメージ（URLは例）
fetch("/api/items")
  .then((res) => res.json())
  .then((data) => console.log(data));
```

> **ポイント**  
> 「POST や更新系は JSON で送ることが多い」程度を押さえ、**詳細はサーバー側の仕様とセット**で学ぶと現場に近づきます。

## まとめ

- 同期は**順番待ち**、非同期は**結果を後から受け取る**イメージ  
- GET は取得、POST は変更系に**使い分け**されることが多い  
- API 連携では **JSON** が主流である
