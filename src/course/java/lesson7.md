# クラスを理解しよう
オブジェクトの設計図である「クラス」からインスタンスを作り、フィールドとメソッドで状態と振る舞いを表現します。

## 本章の目標

本章では以下を目標にして学習します。

- クラスとオブジェクト（インスタンス）の関係を説明できること
- フィールド・メソッド・コンストラクタを定義できること
- <code>new</code> でインスタンスを生成し、フィールドにアクセスできること
- <code>this</code> の役割を説明できること
- 複数インスタンスが独立した状態を持つことを理解すること

## 1. クラスとオブジェクト

**クラス**は設計図、**オブジェクト**（**インスタンス**）はその設計図から作った実体です。

```java
public class Book {
    String title;   // フィールド（状態）
    int price;

    void show() {   // メソッド（振る舞い）
        System.out.println(title + " / " + price + "円");
    }
}
```

```java
Book b1 = new Book();
b1.title = "Java入門";
b1.price = 2200;
b1.show();

Book b2 = new Book();
b2.title = "アルゴリズム";
b2.price = 3000;
b2.show();
```

> **ポイント**
>
> <code>b1</code> と <code>b2</code> は**別々の本**です。<code>title</code> を変えても互いに影響しません（参照が別オブジェクトを指しているため）。

## 2. フィールドとメソッド

| 要素 | 役割 |
| --- | --- |
| フィールド | オブジェクトが持つデータ（インスタンス変数） |
| メソッド | オブジェクトができる操作 |

インスタンスメソッドから、同じインスタンスのフィールドにそのままアクセスできます。

## 3. コンストラクタ

**インスタンス生成時**に呼ばれる特別なメソッドです。名前は**クラス名と同じ**、戻り値は書きません。

```java
public class Person {
    String name;
    int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void introduce() {
        System.out.println(name + "、" + age + "歳です");
    }
}
```

```java
Person p = new Person("花子", 25);
p.introduce();
```

> **ポイント**
>
> コンストラクタでフィールドを初期化しておくと、「未初期化のまま使う」ミスを減らせます。

## 4. this とは

<code>this</code>は「**今のこのインスタンス自身**」を指します。引数名とフィールド名が同じときに区別するために使います。

```java
public Person(String name, int age) {
    this.name = name;  // 左がフィールド、右が引数
    this.age = age;
}
```

## 5. デフォルトコンストラクタ

**コンストラクタを1つも書かない**クラスには、コンパイラが**引数なしのコンストラクタ**を自動で用意します。

```java
public class Point {
    int x;
    int y;
}
// new Point(); が可能
```

**自分でコンストラクタを定義した場合**、引数なしは**自動では付きません**（必要なら明示的に書きます）。

## 6. メソッド内の変数とフィールド

```java
public void example() {
    int local = 10;   // ローカル変数（このメソッドだけ）
    System.out.println(title);  // フィールド
}
```

ローカル変数は**初期化してから**使います。フィールドは型ごとのデフォルト値で初期化されます（例：<code>int</code> は <code>0</code>、参照型は <code>null</code>）。

> **注意**
>
> 参照型フィールドが <code>null</code> のときにメソッドを呼ぶと <code>NullPointerException</code> になります。使う前に代入またはコンストラクタでセットする習慣をつけましょう。

## 7. まとめ

| テーマ | 要点 |
| --- | --- |
| クラス | 設計図。<code>new</code> でインスタンス（オブジェクト）を生成 |
| フィールド | インスタンスごとの状態 |
| メソッド | 振る舞い。インスタンスメソッドからフィールドにアクセス |
| コンストラクタ | 生成時に初期化。<code>this</code> でフィールドと引数を区別 |
| 複数インスタンス | それぞれが独自のフィールド値を持つ |
| ローカル変数 | メソッド内だけ有効。フィールドとは別物 |

> **ポイント**
>
> クラスはこの後の継承・多態性・カプセル化の土台です。「状態はフィールド、操作はメソッド」と分けて考える癖をつけると、設計がしやすくなります。小さな <code>Book</code> や <code>Person</code> を自分で定義して試してみてください。
