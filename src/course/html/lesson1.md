# Webの仕組みとフロントエンドの基本
クライアントとサーバー、HTML/CSS/JavaScript の役割を整理します。

## 本章の目標

本章では以下を目標にして学習します。

- Webアプリの構成（クライアント／サーバー）を図で説明できること
- HTML・CSS・JavaScript それぞれの役割を一言で言えること
- この講義でフロントエンドのどこまでを扱うか把握すること

## 1. Webアプリのざっくりした構成

ユーザーがブラウザでページを開くとき、裏側では次のようなやり取りが起きます。

1. ブラウザ（**クライアント**）がサーバーにリクエストを送る  
2. **サーバー**が HTML などを返す（必要に応じて API でデータも返す）  
3. ブラウザが受け取った内容を表示し、JavaScript が動けば操作に反応する  

> **ポイント**  
> 「全部がブラウザの中だけで完結」と「サーバーと通信する」の境目を意識すると、後の API・非同期の話がつながりやすくなります。

## 2. HTML・CSS・JavaScript の役割

| 技術 | 役割（イメージ） |
| --- | --- |
| **HTML** | 文書の**意味と構造**（見出し、段落、リンクなど） |
| **CSS** | **見た目**（色、余白、レイアウト） |
| **JavaScript** | **動き**（クリックで表示を変える、サーバーと通信するなど） |

## 3. 講義のゴール（フロントの範囲）

**HTML コース**では主に次を扱います（JavaScript は **JavaScript コース**、設計は **設計コース**、TypeScript/React は **React コース**）。

- セマンティックな HTML とアクセシビリティの意識  
- クライアント／サーバーと、CSS・JavaScript との**役割分担の理解**（JS の詳細は別コース）  

> **ポイント**  
> フロントエンドは更新が速い分野です。**「なぜそうするか」**の考え方を一緒に持っておくと、あとから新しいツールが出ても追いやすくなります。

## まとめ

- ブラウザとサーバーが**役割分担**している  
- HTML＝構造、CSS＝見た目、JS＝動き、と整理できる  
- 講義では**基礎とつながり**を重視する
