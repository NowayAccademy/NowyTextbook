# セマンティックタグについて
Webページを作成する際、見た目だけでなく「意味」を正しく伝えることも重要です。  
本章では、HTML5から導入された「セマンティックタグ」について学習します。  

## 本章の目標  

本章では以下を目標にして学習します。  

- セマンティック（意味論的）なマークアップの意義を理解すること  
- 代表的なセマンティックタグ（`<header>`, `<main>`, `<section>`など）の役割を知ること  
- なぜ `<div>` だけでページを作ってはいけないのかを説明できること  

## 1. セマンティックタグとは？  
「セマンティック（Semantic）」とは、「意味を持つ」という意味の言葉です。  
つまり、セマンティックタグとは**そのタグで囲まれた部分が、Webページの中でどのような意味や役割を持っているのか**を、ブラウザや検索エンジンに伝えるためのタグです。  

## 2. `<div>` との違い  
第4章で、`<div>`は「意味を持たない、ひとまとめにするためのタグ」と学びました。  
CSSを使えば、`<div>`だけでも全く同じ見た目のWebページを作ることは可能です。  
しかし、すべてを`<div>`で作ってしまうと、人間が見た目上で区別できても、コンピュータ（検索エンジンや音声読み上げソフト）には「どこがヘッダーで、どこがメインコンテンツなのか」が全く伝わりません。  

```html
<div class="header">サイトのタイトル</div>
<div class="main-content">
  <div class="article">記事の本文</div>
</div>
<div class="footer">コピーライト</div>
```
これをDOMツリーのイメージで図解すると、以下のようになります。  
一番上の`<html>`を頂点として、階層ごとに枝分かれしているのが分かります。  
```html
html
 ├── head
 │    └── title
 └── body
      ├── h1
      └── p
```  
これをセマンティックタグを使って書き換えると、以下のようになります。
```html
<header>サイトのタイトル</header>
<main>
  <article>記事の本文</article>
</main>
<footer>コピーライト</footer>
```  
コードを読む人間にとっても、コンピュータにとっても、構造が劇的に分かりやすくなりました。  

## 3. 代表的なセマンティックタグ  
- `<header>`タグ  
サイトやセクションのヘッダー部分（ロゴやタイトル）を示します。  
- `<nav>`タグ  
主要なナビゲーションリンク（メニュー）のまとまりを示します。  
- `<main>`タグ  
そのページの「メインとなるコンテンツ」を示します。1つのページに1つだけ使用します。  
- `<section>`タグ  
意味的な「章」や「節」の区切りを示します。通常、中に見出し（`<h2>`など）を含みます。  
- `<article>`タグ  
ブログ記事やニュースなど、その部分だけで「独立して完結しているコンテンツ」を示します。  
- `<footer>`タグ  
サイトやセクションのフッター部分（著作権情報や関連リンクなど）を示します。  

## 4. セマンティックタグを使うメリット
デザイン（見た目）が変わらないのに、わざわざ適切なタグを選ぶのには大きな理由があります。  

### 1. SEO（検索エンジン最適化）の向上  
Googleなどの検索エンジンが「このページのどこに重要な情報（メインコンテンツ）が書かれているか」を正確に把握できるようになり、検索結果に適切に反映されやすくなります。  
### 2. アクセシビリティの向上  
視覚に障害がある方が利用する「音声読み上げソフト（スクリーンリーダー）」が、ページ構造を正しく理解し、ユーザーにナビゲーションメニューをスキップさせてメインコンテンツから読み上げる、といったサポートをしやすくなります。  
### 3. 保守性（コードの読みやすさ）の向上  
開発者が後からコードを見たときに、どこに何が書かれているのかを一目で把握しやすくなります。  

> **ポイント**  
>最初から完璧に使い分けるのは難しいかもしれませんが、「ここはヘッダーだから`<header>`にしよう」「ここはナビゲーションだから`<nav>`にしよう」と、意味を考える癖をつけていきましょう。迷った時のグループ化には`<div>`を使って問題ありません。    

## まとめ
- セマンティックタグは、要素の「意味」や「役割」をコンピュータに正しく伝えるためのタグ。  
- `<header>`,`<main>`,`<section>`,`<footer>`などがある。  
- 適切に使用することで、SEO対策やアクセシビリティ（誰もが使いやすいWebサイトであること）の向上につながる。
