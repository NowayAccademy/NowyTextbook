# DOMツリーの概念
WebブラウザがHTMLをどのように解釈しているのか、その裏側の仕組みである「DOMツリー」について学びます。

## 本章の目標

本章では以下を目標にして学習します。  

- DOM（Document Object Model）とは何かを説明できること  
- HTMLが「ツリー構造（木構造）」になっていることを理解すること  
- 親要素、子要素、兄弟要素という関係性を把握すること  

## 1. DOMとは何か？  
DOM（ドム）とは「Document Object Model」の略称です。  
Webブラウザは、私たちが書いたHTMLのコードを読み込むと、それをコンピュータ（JavaScriptなどのプログラム）が理解・操作しやすい形式に変換します。  
この**HTMLをプログラムから操作するための仕組み**をDOMと呼びます。  

## 2. ツリー構造（木構造）  
HTMLは、タグの中に別のタグを入れる「入れ子（ネスト）」というルールで記述していきます。  
この入れ子の構造を細かく分解していくと、ひとつの根元から枝分かれしていく「木」のような形になります。これを**DOMツリー**と呼びます。  

以下のHTMLコードを見てみましょう。  

```html
<html>
  <head>
    <title>DOMの学習</title>
  </head>
  <body>
    <h1>タイトル</h1>
    <p>本文です。</p>
  </body>
</html>
```
これをDOMツリーのイメージで図解すると、以下のようになります。  
一番上の`<html>`を頂点として、階層ごとに枝分かれしているのが分かります。  
```html
html
 ├── head
 │    └── title
 └── body
      ├── h1
      └── p
```

## 3. 親・子・兄弟の関係  
DOMツリーにおける要素同士の関係性は、人間の家族に例えて表現されます。   
この呼び方は、今後CSSやJavaScriptを学ぶ上で頻繁に登場するため、必ず覚えておきましょう。  
- 親要素  
ある要素を一段階外側から囲んでいる要素。  
例：`<title>`にとっての親は`<head>`です  
- 子要素  
ある要素の一段階内側に入っている要素。  
例：`<body>`にとっての子は`<h1>`と`<p>`です。  
- 兄弟要素  
同じ親を持つ要素同士。  
例：`<h1>`と`<p>`は、どちらも`<body>`という同じ親を持つため、兄弟の関係です。  

## 4. なぜDOMツリーを理解する必要があるのか？
HTMLを書くだけであれば、DOMを深く理解していなくても画面に文字を表示することはできます。  
しかし、以下のようなステップに進む際にDOMの概念が必須になります。  

### 1. CSSで思い通りのデザインをするため  
「この親要素の中にある、この子要素だけ色を変えたい」といった指定を行うため。  

### 2. JavaScriptで画面に動きをつけるため  
「ボタン（特定のDOM）がクリックされたら、メニュー（別のDOM）を表示する」といった操作を行うため。  
特に、モダンなフロントエンド開発では、このDOMをいかに効率よく操作するかが重要になります。  
まずは「HTMLは木のような階層構造になっている」というイメージをしっかり持っておきましょう。   

## まとめ
- DOMは、HTMLをプログラムから操作するための仕組みである。  
- HTMLの入れ子構造は、階層的な「DOMツリー」としてブラウザに解釈される。  
- 要素同士の関係は「親要素」「子要素」「兄弟要素」と呼ばれる。  
- 今後CSSやJavaScriptでWebページを操作する上で、ツリー構造の理解が不可欠である。
