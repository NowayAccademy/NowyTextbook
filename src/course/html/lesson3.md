# DOMツリーとセマンティックタグ
ブラウザが作る DOM の親子関係と、意味づけのためのタグを理解します。

## 本章の目標

本章では以下を目標にして学習します。

- DOM ツリー（親子・兄弟）のイメージを説明できること
- `header` / `nav` / `main` などセマンティックタグの役割を述べられること
- SEO・アクセシビリティとの関係をざっくり説明できること

## 1. DOM ツリーとは

ブラウザは HTML を読み込み、**ツリー構造のモデル（DOM）**を内部に持ちます。

- **要素**が**ノード**になる  
- 入れ子の外側が**親**、内側が**子**、同じ階層が**兄弟**  

```html
<body>
  <main>
    <article>
      <h2>記事タイトル</h2>
      <p>本文</p>
    </article>
  </main>
</body>
```

上の例では `main` は `article` の親、`h2` と `p` は兄弟です。

> **ポイント**  
> JavaScript の `document.querySelector` などは、この DOM ツリー上をたどって要素を取得します。

## 2. セマンティックタグ

「このブロックは何の意味か」をタグ名で表します。

| タグ | 意味のイメージ |
| --- | --- |
| `<header>` | ヘッダー（サイト名、ロゴ周りなど） |
| `<nav>` | 主なナビゲーション |
| `<main>` | ページの主たる内容（**1ページに1つ**が推奨） |
| `<article>` | 独立した記事・カードなど |
| `<section>` | テーマごとの区切り |
| `<footer>` | フッター |

## 3. SEO とアクセシビリティ

- **検索エンジン**は見出しの階層や `main` などの構造を手がかりにします。  
- **スクリーンリーダー**はランドマーク（`nav` / `main` など）を利用してジャンプします。  

> **ポイント**  
> 意味のない `div` だけで塗り固めるより、**適切なセマンティックタグ**を選ぶほど、長期的に有利です。

## まとめ

- HTML はブラウザ内で**ツリー（DOM）**として表現される  
- セマンティックタグで**意味**を表すと、機械にも人にも伝わりやすい  
- SEO・支援技術の両方に**良い構造**が効く
