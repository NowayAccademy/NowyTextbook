# 現場のコーディング（3年目エンジニアの視点）
ここまでの章で、Webページをゼロから形にするスキルが身につきました。  
本章では「とりあえず画面に表示される」状態から一歩踏み込み、実務で活躍するエンジニアが必ず意識している「4つの視点（パフォーマンス、UX、SEO、アクセシビリティ）」を学習します。  

## 本章の目標  

本章では以下を目標にして学習します。  

- リンク（`<a>`）とボタン（`<button>`）を正しく使い分けられること  
- スマホ対応（レスポンシブ）に必須のタグを理解すること  
- 画像の遅延読み込みや、入力フォームの補助など、ユーザー体験（UX）を向上させる手法を知ること  

## 1. リンクとボタンの明確な使い分け  
現場のコードレビューで、初心者が最も指摘されやすいのが「`<a>`タグと`<button>`タグの混同」です。  
見た目はCSSでどちらも同じようにデザインできてしまいますが、HTMLの意味としては明確な違いがあります。  

- **`<a>`タグ（リンク）**: 別のURL（ページ）へ移動するときに使用します。  
- **`<button>`タグ（ボタン）**: ページを移動せず、その場で何かしらの処理（データを送信する、ポップアップ画面を開くなど）を行うときに使用します。

```html
<a href="#" class="submit-btn">データを送信</a>

<a href="/about.html">会社概要を見る</a>
<button type="submit">データを送信</button>
```

## 2. レスポンシブ対応の必須設定（Viewport）  
現代のWebサイトは、パソコンよりもスマートフォンで見られることの方が多くなっています。  
スマホの画面サイズに合わせてレイアウトを調整をレスポンシブ対応といいます。  
レスポンシブ対応にするためには、`<head>`内に以下の1行が絶対に必要です。  
```html
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>スマホ対応ページ</title>
</head>
```  
一度ブラウザで開いてみてください。まだ質素ですが、これがすべてのWebページの土台（DOMツリーの根幹）になります。  

## 3. Webパフォーマンスの意識（画像の遅延読み込み）  
Webページの表示速度は、ユーザーの満足度や検索順位（SEO）に直結します。  
画像のデータは重いため、画面に表示されていない下の方の画像まで最初に全て読み込むと、ページの表示が遅くなります。これを防ぐのが`loading="lazy"`という属性です。  

```html
<img src="./images/heavy-photo.jpg" alt="風景写真" loading="lazy">
```  
この属性を付けるだけで、ブラウザが自動的に「スクロールして画像が近づいてきたタイミング」で読み込みを行ってくれます。  
画像を多く使用する場合は設定を検討しましょう。  

## 4. フォームの実践的なUX向上（入力支援）  
ユーザーにとって、フォームの入力は面倒なものです。  
HTMLの属性を少し工夫するだけで、ユーザーの手間を大幅に減らすことができます。  
- `required`属性  
必須項目に設定します。未入力のまま送信しようとすると、ブラウザが自動で警告を出してくれます。
- `autocomplete`属性  
スマホなどのキーボードの「予測変換」を正しく機能させます。
```html
<form>
  <label for="email">メールアドレス（必須）：</label>
  <input type="email" id="email" required autocomplete="email">
  <button type="submit">登録する</button>
</form>
```  

## 5. OGP（SNSでのシェア設定）
URLをX（旧Twitter）などでシェアした際、タイトルや大きな画像がカード状に表示される仕組みを「OGP（Open Graph Protocol）」と呼びます。  
これも <head> 内の <meta> タグで設定します。  
```html
<head>
  <meta property="og:title" content="ページのタイトル">
  <meta property="og:image" content="[https://example.com/images/ogp.jpg](https://example.com/images/ogp.jpg)">
</head>
```  

## まとめ
- 別のページへ飛ぶなら`<a>`、その場で処理をするなら`<button>`を使う。  
- スマホ対応には`<meta name="viewport" ...>`が必須。  
- `loading="lazy"`や`autocomplete`などを活用し、パフォーマンスやユーザーの手間に配慮する。
