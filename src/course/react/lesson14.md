# ポートフォリオ・質疑応答の時間
成果物の相談と、フロントに関する疑問を整理します。

## 本章の目標

本章では以下を目標にして学習します。

- ポートフォリオに載せる題材の**候補と優先順位**を持てること
- 自分の理解度に合わせて**次に学ぶトピック**を列挙できること

## 1. この時間の使い方（想定）

- **ポートフォリオ**の構成・題材・技術選定の相談  
- **フロントエンド全般**の質疑応答  
- キャリアや学習ロードマップの**ざっくり相談**  

> **ポイント**  
> ポートフォリオは「完璧な製品」より**学びと再現性が伝わる**ことが最初は重要です。README に**何を・なぜ・どう動かすか**を書きましょう。

## 2. 題材の例

- 講義で作った UI の**改善版**  
- 好きなドメインの**一覧＋詳細**のミニアプリ  
- 公開 API を使った**検索・一覧**  

## 3. 次のステップ（例）

- React Router、データ取得（SWR / TanStack Query など）  
- テスト（Vitest / React Testing Library）  
- アクセシビリティ、パフォーマンス  

## まとめ

- ポートフォリオは**継続的に磨く前提**で最初の1本を決める  
- 質問は**目的と試したこと**をセットで  
- 学習は**一本道ではない**—興味と求人の両方を見る
