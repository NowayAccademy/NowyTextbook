# コンポーネント思考
UI を部品に分けて組み立てる考え方です。

## 本章の目標

本章では以下を目標にして学習します。

- コンポーネントが**再利用可能なUIの単位**だと説明できること
- 分割のメリット（再利用・テスト・責務の明確化）を述べられること

## 1. 部品に分ける

画面全体を1枚の HTML で書くのではなく、**ヘッダー・カード・ボタン**などに分けます。  
各コンポーネントは**見た目と振る舞いのかたまり**です。

> **ポイント**  
> 「どこまでを1コンポーネントにするか」は経験が必要ですが、最初は**画面の枠**に合わせて粗く切ってよいです。

## 2. 再利用性

同じボタンやカードを**何度も書かず**、props で差分だけ渡します。

## 3. 単方向のデータの土台

親から子へ情報を渡しやすい構造は、後の **props / state** 学習に直結します。

## まとめ

- コンポーネントは**UIのレゴブロック**である  
- 分割で**再利用と保守**がしやすくなる  
- 設計図・ワイヤーと**対応づけ**しやすい
