# JSXとレンダリング
JSX が JavaScript の中でマークアップを書くための糖衣であることを理解します。

## 本章の目標

本章では以下を目標にして学習します。

- JSX が**JavaScript の式**として扱われると説明できること
- HTML との違い（`className` など）の例を挙げられること
- **1ファイルにマークアップとロジックがまとまる**利点を述べられること

## 1. JSX とは

**JavaScript XML** の略で、見た目は HTML に近いですが、実際は **`React.createElement` 呼び出し**に変換されます。

```tsx
function Title() {
  return <h1 className="title">Hello</h1>;
}
```

## 2. HTML との違い（例）

| HTML | JSX（React） |
| --- | --- |
| `class` | `className` |
| `for` | `htmlFor` |

## 3. まとまりのメリット

関連する表示と処理が**同じコンポーネント内**にあり、ファイルを行ったり来たりしにくいです。

> **ポイント**  
> `{}` の中に**式**を埋め込めるのが JSX の強みです（後の条件付きレンダリングに続く）。

## まとめ

- JSX は**見た目に近い構文**で UI を表す  
- 属性名は DOM API に寄せて**一部違う**  
- **関心の近いコード**が1か所に集まる
