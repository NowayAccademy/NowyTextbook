# 型について
型とは何か、なぜ付けるのかを言語非依存寄りに整理します。

## 本章の目標

本章では以下を目標にして学習します。

- 型が**値の種類の約束**だと説明できること
- 型があると**補完・ドキュメント・早期エラー**に効くと述べられること

## 1. 型とは

**その値が何を表し、どんな操作が安全か**の目印です。  
例：数値なら四則演算、文字列なら連結、など。

> **ポイント**  
> 型は「縛り」に見えますが、**チームでの共通言語**にもなります。

## 2. 動的型と静的型（ざっくり）

- **動的** … 実行時に型が決まる／緩い（JS）  
- **静的** … コード上で型を宣言・推論し、**ビルド前に検査**（TS）  

## 3. フロントでの効きどころ

- API のレスポンス形  
- コンポーネントに渡す **props**  
- **state** の形  

## まとめ

- 型は**データの契約**である  
- TS は**開発中のミス**を減らす  
- UI では **props / state** に効く
