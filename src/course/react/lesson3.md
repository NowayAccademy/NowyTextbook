# TypeScriptの基礎：型定義の基本とメリット
コンパイル時エラーに変える考え方と、基本の型注釈です。

## 本章の目標

本章では以下を目標にして学習します。

- 実行時エラーとコンパイル時エラーの違いを説明できること
- `let name: string = "Taro";` のような**基本の型注釈**が書けること

## 1. エラーが出るタイミング

- **実行時** … ブラウザで動かした瞬間に落ちる（ユーザーに見えることも）  
- **コンパイル時** … `tsc` やエディタが**書いている途中**で教えてくれる  

> **ポイント**  
> 「本番で初めて気づく」より「**コミット前に気づく**」方がコストが安いです。

## 2. 基本の型（例）

```typescript
let userName: string = "Taro";
let age: number = 25;
let active: boolean = true;
let items: string[] = ["a", "b"];
```

## 3. 環境

**Node.js** と **TypeScript** のインストールが前提です（事前準備の資料に従ってください）。

## まとめ

- TS は**早い段階**でバグに気づかせる  
- `string` / `number` / `boolean` / 配列型から始める  
- 環境は**事前セットアップ**がスムーズ
