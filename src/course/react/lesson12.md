# 条件付きレンダリング
条件が真のときだけ要素を出し分けます。

## 本章の目標

本章では以下を目標にして学習します。

- `&&` や三項演算子で**表示／非表示**を切り替えられること
- **0 や空文字**が表示に影響する落とし穴に注意できること

## 1. 論理 AND

```tsx
function Banner({ isLoading }: { isLoading: boolean }) {
  return (
    <div>
      {isLoading && <p>読み込み中…</p>}
    </div>
  );
}
```

`isLoading` が `false` のとき、右側は評価されません（React は `false` / `null` をレンダリングしません）。

> **ポイント**  
> **数値の 0** はレンダリングされてしまいます。`count && ...` は `count` が 0 のとき **0 が表示**されるので、`count > 0 && ...` のようにします。

## 2. 三項演算子

```tsx
{ isError ? <p>エラーです</p> : <p>正常です</p> }
```

## 3. 早期 return

```tsx
if (!user) {
  return <p>ログインしてください</p>;
}
return <Dashboard user={user} />;
```

## まとめ

- 条件付きレンダリングは**頻出パターン**である  
- `&&` は**左辺の型**に注意（0 の罠）  
- 三項・早期 return で**読みやすさ**を選ぶ
