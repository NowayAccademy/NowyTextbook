# 導入：TypeScriptとReactの必要性
モダンなフロントが目指すことと、TS・React が補うことです。

## 本章の目標

本章では以下を目標にして学習します。

- 素の JavaScript だけでは**大規模UI**で起きやすい問題を例示できること
- TypeScript が**静的型**で防ぎやすいバグの例を述べられること
- React が**宣言的UI**で何を改善するか説明できること

## 1. なぜモダン開発か

画面が複雑になると、**DOM を直接いじるコード**が散らばり、状態と表示の対応が追いにくくなります。  
また、JavaScript は**実行時まで型エラーに気づけない**ことがあります。

> **ポイント**  
> 「動けばいい」から「**変更容易でチームで回る**」へのステップとして TS とフレームワークが選ばれます。

## 2. TypeScript（TS）

**コンパイル時**に型チェックを行い、**書いている途中**でミスに気づけます。

```typescript
let count: number = 0;
// count = "oops"; // コンパイルエラー
```

## 3. React

**UI を部品（コンポーネント）**に分け、**状態が変わったら表示を再計算する**宣言的な書き方が中心です。  
「どこで DOM を触ったか」より**何を表示すべきか**に集中しやすくなります。

## まとめ

- 規模が大きいほど**型**と**構造化**が効いてくる  
- TS は**早めのエラー検知**  
- React は**コンポーネントと宣言的UI**
