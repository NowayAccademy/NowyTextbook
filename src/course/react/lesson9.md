# State（状態）の概念
時間とともに変わるUIデータをコンポーネントが保持します。

## 本章の目標

本章では以下を目標にして学習します。

- state が**コンポーネント内部の変化するデータ**だと説明できること
- props（親から渡る）との違いを述べられること
- クリック回数・入力値・ローディング表示など**例**を挙げられること

## 1. なぜ state が必要か

props だけでは**ユーザ操作で変わる表示**（カウンタ、入力、開閉状態）を表現しにくいです。  
**state** はコンポーネントが**再レンダリング**のたびに読み取り、更新関数で**次の表示**につなげます。

> **ポイント**  
> 「表示は state と props から**計算結果**として決まる」と考えると、バグが減りやすいです。

## 2. props との違い

| | props | state |
| --- | --- | --- |
| 由来 | 親が渡す | コンポーネント内で保持 |
| 更新 | 親が渡し直す | **専用の更新関数**（次章） |

## 3. 例

- ボタン押下回数  
- テキストボックスの文字列  
- データ取得中フラグ（スピナー表示）  

## まとめ

- state は**変化するUIの源泉**である  
- props は**外からの入力**、state は**内の記憶**  
- 再レンダリングとセットで理解する
