# TypeScriptとReactの統合
`.tsx` の役割と、型を「全部」ではなく「重要なところ」に付ける実務感です。

## 本章の目標

本章では以下を目標にして学習します。

- `.tsx` が**JSX を含む TypeScript**だと説明できること
- 型推論が効くので**すべてに注釈が要らない**と理解すること
- **props と state** を優先的に型で固める姿勢を持てること

## 1. .tsx ファイル

React + TypeScript ではコンポーネントを **`.tsx`** で書きます（純 TS だけなら `.ts`）。

## 2. 型推論

```tsx
const [open, setOpen] = useState(false);
// open は boolean と推論されることが多い
```

初期値から型が分かる場合、**わざわざ `: boolean` と書かない**こともあります。

> **ポイント**  
> 実務では **Props・API境界・複雑な state** に型を厚くし、中間の一時変数は推論に任せる、というバランスが多いです。

## 3. 境界を固める

- コンポーネントの**公開面（props）**  
- サーバーから来るデータの**形**  

ここが崩れると影響が広いので、型・検証を厚めにします。

## まとめ

- `.tsx` で **JSX + 型**  
- 推論を活かし**無駄な注釈を減らす**  
- **境界**の型を優先する
