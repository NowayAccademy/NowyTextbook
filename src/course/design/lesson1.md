# 基本設計（画面レイアウト）とは
画面を文章と図で先に固める「基本設計」の考え方です。

## 本章の目標

本章では以下を目標にして学習します。

- 基本設計書（画面レイアウト）に**何を書くか**説明できること
- 実装前に**画面の枠**を共有する利点を述べられること

## 1. 基本設計とは

**実装の前に**、画面に載せる要素・配置・主要な操作を**文章とワイヤーフレーム**などで固めたものです。  
エンジニアだけでなく、**企画・デザイン・顧客**と認識を合わせるための道具でもあります。

> **ポイント**  
> 「口頭だけ」だと認識ズレが残りやすいです。**図と箇条書き**があると後からの手戻りが減ります。

## 2. 画面レイアウトに含めるとよいこと

- 画面名・URL や画面 ID  
- ヘッダー／フッター／メインエリアの**大枠**  
- フォーム項目、ボタン、一覧、エラー表示の**置き場所**  
- **権限**（ログイン前後で違うか）  

## 3. 実装との関係

フロント実装（React など）に入ると**コンポーネント分割**の単位のヒントになります。  
細部のデザイン（色・フォント）より先に、**情報の塊**を決めるイメージです。

## まとめ

- 基本設計は**実装前の共有物**である  
- レイアウトは**ブロックの配置**から決める  
- 後工程の**手戻り防止**に効く
