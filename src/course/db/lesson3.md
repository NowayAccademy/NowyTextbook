# RDBMS の基本概念
テーブル・レコード・カラム・キーとスキーマを整理します

## 本章の目標
本章では以下を目標にして学習します。

- RDBMS が「表形式でデータを守りながら扱う仕組み」だと説明できること
- テーブル・レコード・カラム・主キー・外部キーの役割を区別できること
- スキーマ（名前空間）のイメージを持てること

## 1. RDBMS とは

**RDBMS（リレーショナルデータベース管理システム）**は、データを**行と列の表（テーブル）**に格納し、**SQL**で操作するためのソフトウェアです。  
PostgreSQL、MySQL、Oracle、SQL Server などが代表例です。

> **ポイント**  
> 「リレーショナル」は、テーブル同士を**キーで関連付けられる**ことに由来します。

## 2. テーブル・レコード・カラム

| 用語 | 意味 |
| --- | --- |
| **テーブル** | 同じ種類のデータを並べた表（例：`customers`） |
| **レコード（行）** | 1件分のデータ（例：ある顧客1人） |
| **カラム（列）** | 属性（例：`name`, `email`, `created_at`） |

### 型（データ型）

各カラムには**型**があり、格納できる値の範囲が決まります（整数、文字列、日時など）。

> **ポイント**  
> アプリの変数と同様に、DB でも**型を間違えると比較・集計で事故**しやすくなります。

## 3. キーの種類

| キー | 役割 |
| --- | --- |
| **主キー（PK）** | 行を一意に特定する。重複・NULL 不可が基本 |
| **外部キー（FK）** | 別テーブルの主キーを参照し、関連を保証する |
| **ユニークキー** | 重複を禁止するが、主キー以外にも使える |

> **ポイント**  
> 主キーは「人間にとっての意味」より、**一意に追えること**が優先されることが多いです（サロゲートキーなど）。

## 4. スキーマとは

**スキーマ**は、テーブルやビューなどオブジェクトをまとめる**名前空間**のようなものです。  
大きなシステムでは `sales` / `inventory` のように分けることもあります。

> **ポイント**  
> 「スキーマ＝テーブル定義」ではありません。**オブジェクトのグループ**と覚えると混乱が減ります。

## 5. 制約のイメージ

- **NOT NULL**：欠損を禁止  
- **CHECK**：値の範囲や形式を限定  
- **DEFAULT**：未指定時の既定値  

制約は、**アプリのバグでもデータを守る**ための最後の砦です。

> **ポイント**  
> 「アプリでチェックすればいい」だけにすると、**バッチや直接SQL**の経路で破綻しやすいです。

## まとめ

- RDBMS は**表＋SQL**でデータを管理する  
- **テーブル／行／列**と**主キー／外部キー**をセットで理解する  
- **スキーマ**は名前空間、**制約**はデータ品質の保険である
