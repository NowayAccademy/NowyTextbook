# 条件検索の実務頻出パターン
LIKE・IN・EXISTS・BETWEEN と注意点を整理します

## 本章の目標
本章では以下を目標にして学習します。

- `LIKE` とワイルドカード（`%` `_`）を使い、**前方一致・部分一致**を書けること
- `IN` と `EXISTS` の使い分けのイメージを持てること
- `BETWEEN` の**端点を含む**挙動と、日付範囲での注意を説明できること

## 1. LIKE（文字列パターン）

```sql
SELECT * FROM customers WHERE name LIKE '山田%';   -- 前方一致
SELECT * FROM customers WHERE email LIKE '%@example.com';
```

- **`%`**：任意長の文字列  
- **`_`**：任意の1文字  

### 注意点

- **先頭 `%`** の部分一致は、インデックスが効きにくく**重くなりやすい**  
- エスケープが必要な場合は `ESCAPE` を使う  

> **ポイント**  
> ユーザー入力をそのまま `LIKE` に埋め込むと**インジェクション**の危険があるため、**プレースホルダ**で渡す。

## 2. IN（値の集合）

```sql
SELECT * FROM orders WHERE status IN ('new', 'paid', 'shipped');
```

`NOT IN` は便利ですが、リスト内に **NULL** があると結果が直感とずれることがあります。

> **ポイント**  
> `IN (サブクエリ)` は行数が多いと重くなることがある。**`EXISTS`** と比較検討する。

## 3. EXISTS（存在チェック）

「関連する行が1行でもあるか」を判定します。

```sql
SELECT c.id, c.name
FROM customers c
WHERE EXISTS (
  SELECT 1 FROM orders o WHERE o.customer_id = c.id
);
```

相関サブクエリとして書くことが多く、**見つかった時点で打ち切れる**ため `IN` より適している場合があります。

> **ポイント**  
> `EXISTS (SELECT 1 ...)` の `1` は慣習で、**存在だけ見る**なら定数でよいです。

## 4. BETWEEN（範囲）

**両端を含む**閉区間です。

```sql
SELECT * FROM orders
WHERE ordered_at BETWEEN '2025-01-01' AND '2025-01-31';
```

日付が `timestamp` のとき、**月末23:59:59まで**など端の扱いをミスしやすいです。  
`>= 月初 AND < 翌月初` の半開区間もよく使われます。

> **ポイント**  
> タイムゾーン付きの型では、**アプリとDBの解釈**がずれると1日ずれる事故が起きます。

## まとめ

- `LIKE` は**パターン検索**だが、**先頭ワイルドは重くなりやすい**  
- `IN` は集合指定、`EXISTS` は**存在判定**に強い  
- `BETWEEN` は**端を含む**。日付範囲は**半開区間**も検討する
