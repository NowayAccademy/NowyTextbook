# 環境構築（A5:SQL Mk-2 と PostgreSQL）
クライアントから接続し、SQLを実行できる状態にします

## 本章の目標
本章では以下を目標にして学習します。

- PostgreSQL をローカル（または指定環境）で起動し、接続できること
- A5:SQL Mk-2（または同等のクライアント）から DB に接続し、SQL を実行できること
- 接続情報（ホスト・ポート・DB名・ユーザー）の意味を説明できること

## 1. 本教材で使う構成

| コンポーネント | 役割 |
| --- | --- |
| **PostgreSQL** | データを格納し、SQL を処理するDBサーバー |
| **A5:SQL Mk-2** | SQL を書いて実行し、結果を表で見るクライアントツール |

> **ポイント**  
> クライアントは **pgAdmin** や **DBeaver** でも構いません。操作名が違うだけで考え方は同じです。

## 2. PostgreSQL のインストールと起動

手順は OS や配布物によって異なります。共通して確認するポイントは次のとおりです。

- **サービスが起動しているか**（Windows のサービス、Docker のコンテナなど）  
- **待ち受けポート**（既定は多くの場合 `5432`）  
- **初期ユーザーとパスワード**（インストーラまたは運用ドキュメントに従う）

> **ポイント**  
> 「インストールできたが接続できない」は、**起動していない・ポートが違う・ファイアウォール**が典型原因です。

## 3. A5:SQL Mk-2 での接続設定

接続ダイアログで入力する主な項目は次のとおりです。

| 項目 | 意味 |
| --- | --- |
| ホスト名 | DB サーバーのアドレス（ローカルなら `localhost` など） |
| ポート | 接続先ポート（例：`5432`） |
| データベース名 | 接続先のDB（例：`postgres` や演習用DB） |
| ユーザー / パスワード | 認証情報 |

接続できたら、**適当な `SELECT` を1つ実行**して結果が返ることを確認します。

```sql
SELECT version();
```

> **ポイント**  
> 接続情報は**機密**です。スクリーンショット共有時はマスクする習慣をつけます。

## 4. 基本操作（最低限）

- **新規SQLエディタ**を開き、文を実行（実行ボタンやショートカット）  
- **結果グリッド**で行数・列名を確認する  
- **エラーメッセージ**を読む（ relation does not exist、permission denied など）

> **ポイント**  
> エラー文は英語でも、**キーワード（テーブル名・列名・権限）**が載っていることが多いです。飛ばさず読みます。

## 5. 演習用スキーマの準備（任意）

講師から `CREATE DATABASE` やダンプリストアの手順がある場合はそれに従います。  
自分で試す場合は、**新しいDBを1つ作り、そこだけで実験する**と安全です。

> **ポイント**  
> 本番相当の接続情報で実験しない。**ローカル／検証環境**を切り分けます。

## まとめ

- PostgreSQL が**起動**し、クライアントから**接続**できることが第一歩である  
- 接続は **ホスト・ポート・DB名・ユーザー** の組み合わせで決まる  
- 実行・結果確認・**エラー読み**を日頃から習慣にする
