# レイアウト手法2（Positionと重ね合わせ）
Flexboxを使えば、ほとんどのレイアウトは作成可能です。  
しかし、「画像の右上に『NEW』というバッジを重ねたい」「スクロールしてもヘッダーを画面上部に固定したい」といった、要素を「浮かせて重ねる」配置には限界があります。  
本章では、自由自在な配置を可能にする `position` プロパティと、重なり順を制御する `z-index` を学習します。  

## 本章の目標  

本章では以下を目標にして学習します。  

- `position: relative;` と `position: absolute;` の「親と子」の連携ルールを理解すること  
- `position: fixed;` と `sticky;` を用いて、要素を画面に追従させられること  
- `z-index` を使って要素の重なり順（前後関係）を制御できること  

## 1. Positionの基本概念と4つの値  
`position` プロパティは、要素の「配置の基準」を決めるものです。  
デフォルト（初期値）は `static`（通常配置）ですが、以下の値に変更することで、要素を上下左右（`top`, `bottom`, `left`, `right`）に動かせるようになります。  

## 2. 絶対配置の黄金コンビ（`relative` と `absolute`）  
実務で最もよく使うのが、要素の上に別の要素をピッタリ重ねるテクニックです。  
ここには**親要素に `relative` を指定し、子要素に `absolute` を指定する**という絶対的なルールがあります。  

- **`position: relative;`（相対配置）**  
  これ単体では見た目は変わりませんが、**子要素が動くときの『基準点（アンカー）』になる**という非常に重要な役割を持ちます。  
- **`position: absolute;`（絶対配置）**  
  指定された要素は通常のレイアウトから「浮き上がり」ます。そして、一番近くにある `relative` が指定された親要素の「左上」を基準（0,0）として、自由な位置に移動します。  

```html
<div class="card">
  <img src="product.jpg" alt="商品">
  <span class="badge">NEW</span>
</div>
```  
```css  
/* 親要素（基準点になる） */
.card {
  position: relative;
}

/* 子要素（浮き上がって移動する） */
.badge {
  position: absolute;
  top: 10px;  /* 親の「上端」から10px下へ */
  right: 10px; /* 親の「右端」から10px左へ */
}
```  
※ もし親要素に`relative`を書き忘れると、子要素は画面の左上（画面全体）を基準にして飛んでいってしまいます。  

## 3. 画面に固定する（fixed と sticky）  
スクロールしても、特定の要素を画面の同じ位置に留めておきたい時に使用します。  
- `position: fixed;`（固定配置）
画面（ウィンドウ）の左上を基準にして要素を完全に固定します。  
「トップへ戻るボタン」や「追従するヘッダー」に使います。  
- `position: sticky;`（粘着配置）
スクロールしてその要素が画面の上端（または指定した位置）に到達した瞬間に、ピタッと固定されるモダンなプロパティです。  
「目次」などでよく使われます。  
```css  
/* スクロールしても画面の右下にずっと固定されるボタン */
.floating-btn {
  position: fixed;
  bottom: 20px;
  right: 20px;
}
```  

## 4. 要素の重なり順を操作する（z-index）  
`position`を使って要素を浮かせると、要素同士が重なることがあります。  
デフォルトでは「HTMLの後に書かれた要素」が上に重なりますが、これを意図的に変更するのが`z-index`です。  
```css  
.modal-window {
  position: fixed; /* 画面に固定（浮かせる） */
  z-index: 100;    /* 数値が大きいほど、手前（上）にくる */
}
```  
`z-index`は、`position: static;`（初期値）の要素には絶対に効きません。  
「z-indexを100にしているのに上にこない！」という時は、必ずその要素に`position: relative;`などの指定がされているか確認してください。  

## まとめ

- position は要素を浮かせて自由に配置するためのプロパティである。  
- 「重ね合わせ」をしたい時は、親に relative、子に absolute を指定する。  
- 画面にずっと固定させたい要素（ヘッダーなど）には fixed を使う。  
- 重なり順を変える z-index は、数値が大きいほど手前に来るが、position が static（初期値）だと効かない。
