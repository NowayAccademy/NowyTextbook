# CSSの役割と記述方法  
HTML編ではWebページの「構造」を作る方法を学びました。  
本コースからは、その構造に対して「見た目（デザイン）」を装飾していくCSSについて学習します。  

## 本章の目標  

本章では以下を目標にして学習します。  

- HTML（構造）とCSS（装飾）を分ける理由を理解すること  
- CSSの基本構文（セレクタ・プロパティ・値）を説明できること  
- CSSを読み込む3つの方法を知り、実務で使われる手法を選択できること  

## 1. CSSとは何か？  
CSSの正式名称は「Cascading Style Sheets」です。  
HTMLで作られた要素に対して、色、サイズ、レイアウトなどの「見た目」を指定するための言語です。  

HTMLの中にデザインの指定を直接書き込むことも技術的には可能ですが、現代のWeb開発では**構造（HTML）と装飾（CSS）を完全に分離する**ことが鉄則です。  
これにより、デザインを一括で変更したり、コードを読みやすく保つ（保守性を高める）ことができます。  

## 2. CSSの基本構文  
CSSは、常に**どこの、何を、どうする**という3つの要素の組み合わせで記述します。  

- **セレクタ（どこの）**：スタイルを適用する対象（HTMLのタグなど）
- **プロパティ（何を）**：変更したいスタイルの種類（文字色など）
- **値（どうする）**：プロパティに対する具体的な設定値

```css
/* h1タグ（セレクタ）の、文字色（プロパティ）を、赤（値）にする */
h1 {
  color: red;
}
```
※ { } の中には複数のプロパティを記述できます。  
各指定の最後には必ずセミコロン（;）をつけます。  

## 3. CSSを適用する3つの方法  
書いたCSSを画面に反映させるには、主に3つのアプローチがあります。  
### 1. 外部ファイルとして読み込む（実務の標準）  
拡張子が .css の専用ファイル（例：style.css）を作成し、HTMLの`<head>`タグ内で読み込みます。  
読み込みは`href`属性にパスを記述します。  
複数のHTMLページから同じCSSを使い回せるため、現場では基本的にこの方法を使用します。  

```css
<head>
  <link rel="stylesheet" href="./css/style.css">
</head>
```

### 2. `<style>`タグ内に書く  
HTMLの`<head>`タグ内に`<style>`タグを用意し、その中にCSSを記述します。  
そのページにしか適用されないため、ページ数が少ない簡素なサイトでのみ使われます。  

```html
<head>
  <style>
    p { color: blue; }
  </style>
</head>
```  

### 3. インラインで直接書く（非推奨）  
HTMLのタグに直接 style 属性を追加して記述します。  
HTML（構造）とCSS（装飾）が混ざってしまい管理が難しくなるため、通常のコーディングでは原則として使用しません。  

```html
<h1 style="color: red;">NOWAYカフェへようこそ</h1>
```

> **ポイント**  
> 初心者のうちは「とりあえず画面が変わればOK」と思いがちですが、現場では「数ヶ月後に別のエンジニアが見ても修正しやすいか（保守性）」が問われます。  
CSSを外部ファイルに分けるのも、保守性を高めるための第一歩です。

## まとめ

- CSSはWebページの「見た目」を制御する言語である  
- 構造（HTML）と装飾（CSS）を分離することで保守性が高まる  
- CSSは「セレクタ { プロパティ: 値; }」という構文で記述する  
- 外部ファイル（.css）を作成し、HTMLの`<link>`タグで読み込む
