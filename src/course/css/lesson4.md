# セレクタ
実務に入ると「CSSを書いたのにデザインが変わらない！」という壁に必ずぶつかります。  
本章では、より高度な要素の指定方法セレクタと、CSSが適用される優先順位を学習します。  

## 本章の目標  

本章では以下を目標にして学習します。  

- クラス（`.`）とID（`#`）の違いを理解し、実務でクラスを優先する理由を説明できること  
- 複数のセレクタを組み合わせて、特定の要素だけを狙い撃ちできること  
- マウスカーソルが乗った時（`:hover`）などの疑似クラスを使えること  
- CSSが効かない時の原因である「詳細度」を理解できること  

## 1. クラスとIDの使い分け  
特定の要素だけにスタイルを当てるための記述方法です。  

- **クラス（`.`）**: 1つのページ内で**何度も使える**グループ分けの目印。CSSでは先頭にドット（`.`）をつけます。  
- **ID（`#`）**: 1つのページ内で**1度しか使えない**固有の目印。CSSでは先頭にハッシュ（`#`）をつけます。  

```css
/* クラスの指定方法（99%こちらを使います） */
.button {
  background-color: blue;
}

/* IDの指定方法（CSSの装飾に使うのは非推奨です） */
#header-logo {
  width: 200px;
}
```  
### IDでCSSを書かない
IDはページ内に1つしか存在してはいけないため、「同じデザインのボタンをもう一つ作りたい」といった使い回し（コンポーネント化）ができません。  
CSSの装飾は特別な理由がない限り「クラス」で行うのが鉄則です。  

## 2. セレクタの組み合わせ  
HTMLのツリー構造を利用して、より細かく要素を指定する方法があります。  

- ### 子孫セレクタ  
ある要素の中にある要素を指定します。
```css  
/* .card の中にある h2 だけを赤にする（外にあるh2には影響しない） */
.card h2 {
  color: red;
}
```  

- ### 複数クラスの指定  
2つのクラスを両方持っている要素だけを指定します。
```css  
/* .btn と .btn-danger を両方持っている要素 */
.btn.btn-danger {
  background-color: red;
}
```

## 3. 疑似クラス  
ユーザーの操作や、要素の順番に応じた指定を行う便利なセレクタです。  
```css  
/* マウスカーソルが乗った時だけ色を変える（よく使います） */
.button:hover {
  background-color: darkblue;
}

/* リスト（li）の一番最初の要素だけ上の線を消す */
li:first-child {
  border-top: none;
}
```  

## 4. 詳細度とは  
CSSには「下にある記述ほど上書きされて強くなる」という基本ルールがありますが、それよりも優先される**詳細度**という強力なルールが存在します。  
以下のHTMLを例に見てみましょう。  
```css  
<p class="text-red">この文字は何色になるでしょう？</p>
```  

```css  
/* 詳細度によって上のredが反映されてしまう */
.text-red {
  color: red; 
}

p {
  color: blue; 
}
```  
結果は赤色になります。下に書かれた p よりも、上に書かれた .text-red の方が詳細度が高いためです。  
「CSSを書いたのに反映されない！」という時は詳細度を見直しましょう。  

## まとめ

- CSSは、使い回しがしやすいクラス（.）で指定する。
- 空白を使った「子孫セレクタ」や、状態を表す :hover などを活用する。
- CSSには詳細度があるので、優先順位される。
