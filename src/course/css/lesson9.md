# 現場のCSS（リセットCSS・保守性・ライブラリ）
ここまでの学習で、Webページを思い通りにデザインする力が身につきました。  
しかし、プロの開発現場では「見た目が綺麗に作れる」ことはゴールではなくスタートラインです。  
本章では、数ヶ月後に別の人がコードを見ても迷わず修正できる「保守性の高いCSS」の書き方と、現代の開発を支える強力なツール群について学習します。  

## 本章の目標  

本章では以下を目標にして学習します。  

- ブラウザ間の表示のズレをなくす「リセットCSS」の役割を理解すること  
- CSSが破綻する原因を知り、規則性のあるクラス名（命名規則）の重要性を説明できること  
- Sass、Bootstrap、Tailwind CSSなど、現場で使われるツールの特徴と違いを知ること  

## 1. ブラウザの差異をなくす「リセットCSS」  
実は、Google ChromeやSafariといったWebブラウザには、最初から「デフォルトのCSS」が設定されています（例：`<h1>`にはあらかじめ大きな余白がある、など）。  
厄介なことに、このデフォルト設定はブラウザごとに微妙に異なります。そのため、自分が書いたCSSがChromeでは綺麗に見えても、Safariではズレてしまうという現象が起こります。  

これを解決するのが**リセットCSS**です。  
すべてのブラウザの余計なデフォルト設定を一旦「ゼロ」にするための共通のCSSファイルです。  
現場では、自分でCSSを書く前に、必ずこのリセットCSS（またはブラウザ間の差異を整えるNormalize.css）を最初に読み込みます。  

```html
<head>
  <link rel="stylesheet" href="reset.css">
  <link rel="stylesheet" href="style.css">
</head>
```

## 2. 破綻しないクラス名と「BEM」の概念  
CSS最大の弱点は**すべての指定がグローバル（全体）に影響してしまうこと**です。  
例えば、自己紹介ページで「タイトルだから」と`.title`というクラス名をつけると、  
後から別のページで「ここのタイトルも`.title`でいいや」と書いた瞬間、前のページのデザインが崩れるという事故が多発します。  
これを「CSSが破綻する」と言います。   
プロのエンジニアは、予測不能な影響を減らし、安全にコードを管理するために**命名規則」というルールに従ってクラス名をつけます。**  
世界的に有名なルールの1つがBEM（ベム）です。  

BEMの基本的な考え方（Block, Element, Modifier）  
塊（Block）、その中の要素（Element）、状態（Modifier）をアンダースコア（__）やハイフン（--）で繋いで、誰が見ても役割がわかるように名前をつけます。  
```css
/* 悪い例：どこに影響するか分からない */
.title { color: red; }
.active { background: blue; }

/* 良い例（BEM風）：カード（Block）の中の、タイトル（Element）であることが一目でわかる */
.card__title { color: red; }

/* 良い例（BEM風）：ボタン（Block）の、アクティブな状態（Modifier）であることがわかる */
.btn--active { background: blue; }
```  
最初から完璧なBEMを書く必要はありませんが、「他の場所と名前が被らないか？」「数ヶ月後の自分が見て、どこの装飾か瞬時に理解できるか？」を意識することが、プロへの第一歩です。  

## 3. 開発を加速させるモダンなツール群  
素のCSSをマスターした先には、より効率的に開発を行うためのツールが待っています。  
現場に出る前に、名前と役割だけでも知っておきましょう。  
- ### コンポーネント指向CSSフレームワーク（Bootstrap）  
世界で最も高いシェアを誇り、数多くの既存プロジェクトや業務システムで採用されているのが「Bootstrap（ブートストラップ）」です。  
「すでにデザインが完成しているボタンやメニュー（コンポーネント）」を丸ごと提供してくれるため、デザインに悩む時間をゼロにして爆速で画面を作ることができます。  
```html  
<button class="btn btn-primary">送信する</button>
```  
「どのサイトも似たようなデザイン（Bootstrap臭）になりがち」という特徴もありますが、現場に配属された際、遭遇する確率が極めて高い重要なツールです。  
- ### ユーティリティファーストなCSSフレームワーク（Tailwind CSS）  
現代のフロントエンド開発（特にReactなどのモダンな環境）において、新たな主流となりつつあるのが「Tailwind CSS」です。  
Bootstrapとは対照的に、あらかじめ flex や text-center、mt-4（margin-top: 1remの意味）といった「1つの役割だけを持つ細かいクラス」が大量に用意されており、HTMLにそれを組み合わせるだけでデザインを作ります。  
```html  
<div class="flex items-center justify-between p-4 bg-white rounded-lg shadow">
  <h2 class="text-xl font-bold text-blue-600">Tailwind CSSの例</h2>
</div>
```  
CSSファイルを切り替える視線移動がなくなり、命名規則（クラス名）に悩む必要もなくなるため、開発スピードと保守性が劇的に向上します。  
BootstrapやTailwind CSSなどの便利なツールも、裏側で動いているのはこれまで学んできた`display: flex;`や`box-sizing: border-box;`といった「CSSの基礎知識」です。  
基礎がなければ、これらの便利なツールを現場で使いこなすことはできません。  

## まとめ

- 開発を始める前に「リセットCSS」を読み込み、ブラウザ間の表示の差異をなくす。  
- CSSの破綻を防ぐため、BEMなどの命名規則を意識して「被らない・分かりやすいクラス名」をつける。  
- 現場では記述を効率化するSassや、圧倒的なシェアを持つBootstrap、モダンなTailwind CSSなどのツールが活用されている。  
- すべての便利ツールの土台には、本コースで学んだ「HTML/CSSの基礎」が必ず存在している。
